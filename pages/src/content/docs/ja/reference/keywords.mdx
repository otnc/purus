## 宣言

| キーワード | JS出力 | 説明 |
|---|---|---|
| `const` | `const` | 定数宣言 |
| `let` | `let` | 変数宣言 |
| `var` | `var` | var宣言（非推奨） |
| `be` | `=` | 代入 / 再代入（`const`・`let`・`var` またはプロパティ・インデックス対象の後ろに必要） |

:::caution[裸の変数代入はサポートされていません（v0.10.x以降）]
宣言キーワードなしに裸の識別子に `be` を使用するとコンパイルエラーになります。必ず `const`・`let`・`var` で宣言してください:

```purus
-- エラー
x be 42

-- OK
const x be 42
let x be 42

-- OK（プロパティ・インデックス対象に宣言キーワードは不要）
obj.prop be value
arr[\0] be value
```
:::

## 文字列

| 構文 | 説明 |
|---|---|
| `//;text;//` | 文字列リテラル — **推奨** |
| `///text///` | 文字列リテラル — 使用可能（元の構文） |
| `//;text [expr] text;//` | 補完付き文字列（推奨形式） |
| `///text [expr] text///` | 補完付き文字列（元の形式） |

どちらの構文も同一のJavaScript出力を生成します。`//;...;//` が推奨される理由：`;` デリミタは `[expr]` 補完と競合せず、URLなど内容に `//` が含まれても終端 `;//` と視覚的に混乱しません。

## 記号

| 記号 | 意味 |
|---|---|
| `[` `]` | ブラケット（呼び出し、配列、オブジェクト、グループ化） |
| `.` | プロパティアクセス |
| `\.` | オプショナルチェイニング（`?.`） |
| `\` | 計算アクセスプレフィックス（`[...]` 内で使用） |
| `;` | 引数 / 要素の区切り |
| `,` | 配列 / オブジェクトの区切り |
| `..` | 包含範囲 |
| `...` | 排他範囲 |
